# Установка
если вы **РЕАЛЬНО** не хотите использовать Docker или у вас нет видеокарты Nvidia(не факт что обычная установка не прокатит при ее отсутствии), делайте следующее:
1. клонируйте эту ветку в папку /ulstu/ так, чтобы ПОЛНЫЙ путь до, например, ros2_ws/ выглядел ТОЧНО ТАК ЖЕ(это хардкод со стороны разрабов УлГТУ с расчетом на работу в контейнере, исправлят мне лень):  
   ![image](https://github.com/user-attachments/assets/f5bff6fb-bc0d-4936-865a-20991dc7898a)  
и `tree -L 2/ulstu/` должно вернуть  
![image](https://github.com/user-attachments/assets/3bc8b2d5-7fbd-49f7-a820-2df49d49971f)
2. выполните этот код, чтобы установить ros2 и python пакеты, модули хз 
```bash
cd /ulstu/
sudo apt install $(cat pkg.txt)
pip install -r requirements.txt
```
3. наслаждайтесь)
# ВАЖНОЕ УТОЧНЕНИЕ  
я пробовал установку всего этого дела на ```Ubuntu 22.04```, ```Ubuntu 24.04```. На последнем, насколько я понял, ```AppArmor``` блокирует контейнерам доступ к консоли, мб можно исправить, но разработчики ```ROS2``` обещают стабильную работу только на версиях не младше 22.04, поэтому у меня и ```Docker``` контейнер, и no-gpu (эта) установка работают стабильно(почти) на 22.04.

## не очень важное уточнение
при запуске КАЖДОЙ инстанции терминала необходимо прогонять ```source install/setup.bash``` , тк этот скрипт устанавливает переменные среды, НО  
у меня не всегда работает эта команда, поэтому я использую  
```
source /opt/ros/humble/setup.bash
source ~/ros2_humble/install/setup.bash 
```  
если что-то отсутствует  
### надеюсь не пригодится  
   Вся работа в этой ветке может мерджиться автоматически с `main` за исключением запускающих файлов, файлов с абсолютными путями или при установке дополнительных пакетов(любых, кроме утилит для системы)
